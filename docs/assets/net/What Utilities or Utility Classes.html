What Utilities or Utility Classes?
原文
A utility class is one which:
  • Has no state (fields) of its own, so all the methods can be class methods (static) rather than object methods (requiring an instantiation of an object)
  • Provides methods for multiple other classes (shared code)
實用程序類是：
  • 沒有自己的狀態（字段），因此所有方法都可以是類方法（靜態）而不是對象方法（需要實例化對象）
  • 為多個其他類提供方法（共享代碼）
翻譯
實用程序類是：
  • 沒有自己的狀態（字段），因此所有方法都可以是類方法（靜態）而不是對象方法（需要實例化對象）
  • 為多個其他類提供方法（共享代碼）
它們在Java等沒有多重繼承的語言中很有用。在像C ++這樣具有多重繼承的語言中，你通常會將共享代碼放在一個超類中，它可以被任何使用它的類繼承; 在Java 8及更高版本中，您可以將靜態方法放在類似的接口上。
因為實用程序類沒有字段，並且沒有實例化，所以它們本身不是面向對象的。但是生成易於調試/維護代碼比嚴格遵守OO設計原則更重要，因此如果實用程序類最有意義，我會說這樣做。有關減少使用實用程序類的危險的一些最佳實踐，請參閱我在Java中編寫實用程序類的答案。

C# sealed vs Java final()

另外再尋找文章時，看到了大家都說 Java 的 Utility Classes 類別上建議加上final ，所以特別去查詢了一下，.net相對應的設定方式。
原文
hat's because final in Java means plenty of different things depending on where you use it whereas sealed in C# applies only to classes and potentially virtual members (methods, properties, events).
In Java final can be applied to:
  • classes, which means that the class cannot be inherited. This is the equivalent of C#'s sealed.
  • methods, which means that the method cannot be overridden in a derived class. This is the default in C#, unless you declare a method as virtual and in a derived class this can be prevented for further derived classes with sealed again.
  • fields and variables, which means that they can only be initialized once. For fields the equivalent in C# is readonly.
翻譯
這是因為final在Java中意味著很多不同的東西取決於你使用它的位置，而sealed在C＃中只適用於類和潛在的虛擬成員（方法，屬性，事件）。
在Java final中可以應用於：
  • 類，這意味著該類不能被繼承。這相當於C＃的sealed。
  • 方法，這意味著無法在派生類中重寫該方法。這是C＃中的默認值，除非您將方法聲明virtual為派生類，並且可以sealed再次阻止此更多派生類。
  • 字段和變量，這意味著它們只能初始化一次。對於C＃中的等效字段readonly。
